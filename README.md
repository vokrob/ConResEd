# ConResEd

Конвертер и редактор резюме

## Что сейчас есть
- Backend на Flask
- Frontend на React
- Регистрация и авторизация пользователя
- PostgreSQL с инициализацией тестовых данных
- Страницы с шаблонами резюме
- Сохранение информации в заполненных полях при переходах между шаблонами
- Генерация QR-кодов для быстрого доступа к резюме с мобильных устройств

## Запуск
```bash
docker compose up --build
```

После запуска: [http://localhost:8080](http://localhost:8080)

## Доступ с мобильных устройств (через QR-код)

Для того чтобы QR-код открывался на телефоне без ручного изменения URL, нужно настроить переменную окружения `VITE_APP_URL`:

### Вариант 1: Использование переменной окружения (рекомендуется)

1. Узнайте ваш локальный IP-адрес в сети:
   - **Windows**: выполните `ipconfig` в командной строке, найдите IPv4 адрес (например, `192.168.1.100`)
   - **macOS/Linux**: выполните `ifconfig` или `ip addr`, найдите адрес вашей Wi-Fi сети

2. Создайте файл `.env` в папке `frontend/`:
   ```bash
   cd frontend
   echo "VITE_APP_URL=http://192.168.x.x:8080" > .env
   ```
   Замените `192.168.x.x` на ваш реальный IP-адрес.

3. Перезапустите проект:
   ```bash
   docker compose up --build
   ```

Теперь QR-коды будут генерироваться с правильным URL для доступа из локальной сети.

### Вариант 2: Запуск Vite dev server с доступом из сети

Если вы разрабатываете без Docker, добавьте в `vite.config.js`:
```js
server: {
  host: true, // Разрешает доступ с других устройств
}
```

И используйте URL вида `http://192.168.x.x:5173` на мобильных устройствах.

### Вариант 3: Использование туннеля (ngrok, localtunnel)

Для временного публичного доступа можно использовать сервисы вроде ngrok:
```bash
ngrok http 8080
```
Полученный публичный URL укажите в `VITE_APP_URL`.
