# Сценарий выступления (до 10 минут)

## 1) Зачем проект (1-2 минуты)
- Мы делаем ConResEd, чтобы сократить время на подготовку резюме.
- Целевая аудитория: студенты и junior-специалисты.
- Проблема: у пользователей есть содержимое резюме, но нет удобного, быстрого оформления.

## 2) Что уже сделано в PoC (2-3 минуты)
- Есть frontend, backend и PostgreSQL.
- Реализована интеграция UI -> Backend -> DB.
- Проект разворачивается в dev-режиме одной командой через Docker.

## 3) Что вошло в MVP-минимум текущего этапа и почему (1-2 минуты)
- Взяли узкий, но проверяемый пользовательский путь: "клик в UI -> ответ из БД".
- Это подтверждает техническую реализуемость архитектуры перед развитием функций редактора резюме.

## 4) Live-демо прототипа (2-3 минуты)
1. Показать команду: `docker compose up --build`.
2. Открыть `http://localhost:8080`.
3. Нажать кнопку получения данных.
4. Показать ответ и endpoint здоровья `http://localhost:5000/api/health`.

## 5) План следующего релиза (1-2 минуты)
- Добавить полноценное заполнение резюме в шаблоне.
- Добавить несколько шаблонов и предпросмотр.
- Реализовать экспорт в PDF.
- Добавить пользовательские сценарии хранения нескольких версий резюме.
