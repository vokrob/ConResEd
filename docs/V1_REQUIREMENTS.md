# V1 - общие требования (исправленная версия)

## Цель продукта
ConResEd помогает пользователю быстро создать аккуратно оформленное резюме из существующего файла или с нуля, используя готовые шаблоны.

## Целевая аудитория
- Студенты и начинающие специалисты
- Кандидаты, которым нужно быстро адаптировать резюме под разные вакансии

## Функциональные требования
1. **Импорт исходного резюме**
   - Пользователь загружает текстовое резюме.
   - Система отображает его рядом с шаблоном для ручного переноса данных.

2. **Редактор резюме**
   - Пользователь выбирает шаблон.
   - Пользователь заполняет/редактирует поля резюме в интерфейсе.

3. **Библиотека шаблонов**
   - Доступно 3-5 шаблонов.
   - Есть предварительный просмотр перед применением.

4. **Сохранение и версии**
   - Резюме сохраняется в профиле пользователя.
   - Можно хранить несколько версий резюме.

5. **Экспорт**
   - Экспорт в PDF с сохранением структуры и стилей.

## Нефункциональные требования
- Время отклика UI при базовых действиях: до 2 секунд.
- Поддержка запуска в dev-окружении: frontend + backend + PostgreSQL.
- Базовая устойчивость к ошибкам (понятные сообщения в UI, healthcheck backend).

## Критерии готовности PoC
- Работает сквозной поток: UI -> Backend -> DB.
- Демонстрируется разворачивание окружения в dev-режиме.
- Подготовлен сценарий демонстрации для подгруппы.
