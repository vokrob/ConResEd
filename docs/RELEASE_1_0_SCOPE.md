# release/1.0 - вырезка требований под PoC

## Что входит в релиз
1. Запуск dev-окружения одной командой (`docker compose up --build`):
   - Frontend
   - Backend
   - PostgreSQL

2. Сквозной сценарий демонстрации:
   - Пользователь открывает frontend.
   - Нажимает кнопку получения данных.
   - Frontend обращается к backend.
   - Backend читает данные из PostgreSQL.
   - Пользователь видит успешный результат.

3. Техническая наблюдаемость для демо:
   - Endpoint проверки здоровья: `GET /api/health`.

## Что не входит в релиз 1.0 (перенос в следующий релиз)
- Полный редактор всех разделов резюме.
- Полноценная библиотека из 3-5 визуальных шаблонов.
- Авторизация и личный кабинет.
- Экспорт в PDF.
- Автосохранение и продвинутое версионирование резюме.
